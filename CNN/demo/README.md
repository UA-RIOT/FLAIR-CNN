# FLAIR-CNN Live Demo — Setup & Run Instructions

**CNN Autoencoder for Network Intrusion Detection**
University of Arkansas — RIOT Lab

---

## What the demo shows

An interactive Streamlit app that runs the INT8-quantized CNN autoencoder on
individual windows from the WUSTL-IIoT dataset and visualizes the detection decision.

Three steps are shown for each window:

1. **Input Window** — heatmap of the 10 flows × 21 normalized features fed to the model,
   plus a table of the decoded categorical features (Sport / Dport / Proto).
2. **CNN Reconstruction** — what the decoder produced vs. what went in, plus a
   per-feature error bar chart showing which features were hardest to reconstruct.
3. **Anomaly Score & Classification** — a gauge comparing the MSE score to the
   99th-percentile threshold, with the ATTACK / NORMAL decision highlighted.

Use the sidebar filter (All / Normal Only / Attack Only) and the window slider to
explore any of the ~1.19M windows in the dataset.

## Environment setup

The demo uses a **dedicated lightweight conda environment** (`cnn-demo`) with
only what Streamlit needs. This keeps the Ryzen AI quantization environment
(`ryzen-ai-1.7.0`) clean.

### Create the environment (once)

From the **project root**, with any conda active:

```bash
conda env create -f CNN/demo/environment.yml
```

This installs Python 3.11, Streamlit, Plotly, standard onnxruntime (CPU), NumPy,
and Pandas. Total size is roughly 500 MB.

### Activate the environment

```bash
conda activate cnn-demo
```

Your prompt should show `(cnn-demo)` when active.

---

## Running the demo

From the **project root** with `cnn-demo` activated:

```bash
streamlit run CNN/demo/app.py
```

Streamlit prints a local URL — open it in your browser:

```
Local URL: http://localhost:8501
```

The app loads in **2–5 seconds** (ONNX session warm-up + dataset memory-map).

### Full copy-paste sequence

```bash
conda activate cnn-demo
streamlit run CNN/demo/app.py
```

### Run on a different port (if 8501 is taken)

```bash
streamlit run CNN/demo/app.py --server.port 8502
```

### Stop the demo

Press `Ctrl + C` in the terminal.

## How it works

### At startup (runs once)

1. **ONNX session created** from `cnn_quantized.onnx` using the standard CPU
   execution provider. The Ryzen AI NPU is not used here — CPU is fast enough
   for single-window inference in an interactive demo, and avoids requiring the
   full VitisAI driver stack on the demo machine.

2. **Dataset memory-mapped** from `preprocessed.npz`. The full ~1.19M-window
   array is never loaded into RAM — the OS pages in only the slice you access.

3. **Threshold loaded** from `cnn_deploy_meta.npz`. This was computed on the
   training machine as the 99th percentile of held-out normal-window scores.

### On every slider move (~5–20 ms)

1. One window sliced from the memory-mapped arrays (~2 KB read from disk).
2. Single ONNX forward pass: embed → Conv1d encoder → Conv1d decoder → reconstruction.
3. Anomaly score computed as mean MSE across all features and timesteps.
4. All three visualizations rendered live from the forward pass outputs.

### Why CPU instead of NPU for the demo

The NPU path requires the full Ryzen AI driver stack and the `VitisAIExecutionProvider`
build of onnxruntime. A single-window forward pass on CPU takes ~5–15 ms, which is
perfectly responsive for an interactive slider. Keeping the demo on CPU makes it
portable — it runs on any machine that has the conda env, without needing a Strix
NPU or the Ryzen AI SDK installed.

## Troubleshooting

**`streamlit: command not found`**
→ The `cnn-demo` environment is not active. Run `conda activate cnn-demo`.

**`ModuleNotFoundError: No module named 'onnxruntime'`**
→ The environment was not created correctly. Run `conda env create -f CNN/demo/environment.yml` again.

**`FileNotFoundError: cnn_quantized.onnx`**
→ Run `python CNN/quantize_cnn.py` first.

**`FileNotFoundError: cnn_deploy_meta.npz`**
→ Copy this file from the training machine. It is generated by `python CNN/evaluate_cnn.py`.

**`FileNotFoundError: preprocessed.npz`**
→ Run `python scripts/preprocess_data.py` first.

**Port already in use**
→ Run `streamlit run CNN/demo/app.py --server.port 8502`.

**App is very slow on the slider**
→ This is normal the first time a region of the dataset is accessed (disk page-in).
  Repeated accesses to the same region will be fast (OS cache).
